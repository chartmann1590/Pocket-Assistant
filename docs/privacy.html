Privacy Policy

Effective date: April 12, 2026 · Last updated: April 12, 2026

Pocket Assistant ("the app") is a local-first AI organizer for Android. This policy explains what data the app handles, how it is processed, and what choices you have.

1. Data we collect

None.

Pocket Assistant has no backend server, no analytics service, and no user accounts. We do not collect, transmit, or store any personal data. All content you import — screenshots, photos, PDFs, and pasted text — stays in a local database on your device.

2. On-device processing

The following processing happens entirely on your device:

  • OCR — ML Kit text recognition runs on-device to extract text from images and PDFs.
  • AI classification and summarization — a downloaded LiteRT-LM model runs locally. No data is sent to a cloud API.
  • Semantic search — MediaPipe Universal Sentence Encoder computes embeddings on-device.
  • Entity extraction — ML Kit entity extraction identifies dates, amounts, and contacts locally.
  • Tasks and reminders — scheduled via the Android AlarmManager on your phone.

3. Optional network use

The app accesses the internet only in these explicit, user-initiated scenarios:

  • Local model download — when you choose to download an on-device LLM model, the app fetches it from Hugging Face. Only the model file is transferred; no user data is sent.
  • Ollama integration — if you configure an Ollama server in Settings, prompts and context are sent to the endpoint you specified. This is entirely opt-in; the app does not contact any server unless you provide a URL.
  • Update checker — on launch the app queries the GitHub Releases API for your repository to check whether a newer build is available. No personal data is included in this request.

4. Permissions

  • Camera / photo access — used only when you import a photo or scan a document.
  • Notifications — used only to deliver reminders you scheduled.
  • Internet — used for model download, Ollama (if configured), and update checks.

No permission is used for advertising, tracking, or analytics.

5. Third-party services

The open-source build of Pocket Assistant does not include any third-party analytics, crash reporting, or advertising SDKs. If a build variant enables ads (via the ADS_ENABLED flag), Google AdMob may collect data according to Google's Privacy Policy. The default open-source build ships with ads disabled.

6. Children's privacy

Pocket Assistant is not directed at children under 13. We do not knowingly collect personal information from children.

7. Data retention and deletion

All data is stored locally on your device. You can delete everything at any time via Settings → Local Data → Clear database, or by uninstalling the app. There is nothing to delete on our end because we never receive your data.

8. Changes to this policy

If we update this policy, the new version will be published at this URL and the "Last updated" date above will change. Material changes will be noted in the app's release notes.

9. Contact

Questions or concerns? Open an issue on the GitHub repository.
